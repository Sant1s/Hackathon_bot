# Swagger API Документация

## Доступ к Swagger UI

После запуска сервера, Swagger UI будет доступен по адресу:

```
http://localhost:8080/swagger/index.html
```

## Регенерация документации

Если вы изменили аннотации в handlers, необходимо регенерировать документацию:

```bash
swag init -g main.go -o ./docs
```

Или если swag не установлен глобально:

```bash
go install github.com/swaggo/swag/cmd/swag@latest
swag init -g main.go -o ./docs
```

## Использование

1. Запустите сервер: `go run .`
2. Откройте браузер и перейдите на `http://localhost:8080/swagger/index.html`
3. В Swagger UI вы можете:
   - Просматривать все доступные endpoints
   - Тестировать API прямо из браузера
   - Авторизоваться используя кнопку "Authorize" (введите токен в формате: `Bearer <your_token>`)
   - Просматривать схемы данных и примеры запросов/ответов

## Аутентификация в Swagger

Для тестирования защищенных endpoints:

1. Сначала выполните `/auth/login` или `/auth/register` чтобы получить JWT токен
2. Нажмите кнопку "Authorize" в правом верхнем углу Swagger UI
3. Введите токен в формате: `Bearer <ваш_токен>`
4. Теперь все защищенные endpoints будут использовать этот токен

## Структура документации

Документация организована по тегам:

- **Аутентификация** - регистрация, вход, обновление токена
- **Профиль** - управление профилем пользователя
- **Верификация** - заявки на верификацию
- **Посты** - управление постами о помощи
- **Пожертвования** - создание и управление пожертвованиями
- **Чаты** - обмен сообщениями
- **Рейтинг** - рейтинг пользователей
- **Утилиты** - вспомогательные endpoints
